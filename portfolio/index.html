Scenario Planning MVP

A reproducible scenario-analysis pipeline that models demand/cost shocks and publishes auditable run artifacts using Airflow, MinIO, and Postgres.

Like this project? The full menu of nerdy builds lives on my website: manoj-nawale.github.io — more projects, same chaos-control energy.

Problem it solves

Planning teams often need fast what-if analysis, but ad-hoc notebook workflows make runs hard to reproduce and compare. This project operationalizes scenario simulation into an orchestrated pipeline with consistent run metadata and artifact structure.

How it works

  • Generates deterministic baseline input data (seeded synthetic dataset)
  • Applies scenario knobs: demand, variable cost, and logistics cost scaling
  • Runs a toy capacity-aware solver with overtime premium handling
  • Publishes inputs/outputs to MinIO partitioned by dt/run_id/scenario_id

How to run

docker compose up -d
# open http://localhost:8080 (admin/admin)
# trigger DAGs:
#   hello_scenario_planning (optional)
#   minio_smoketest (optional)
#   scenario_planning_mvp (main)
# validate MinIO outputs at http://localhost:9001 (bucket: scenarioops)
